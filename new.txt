  Key translation notes:
  - UUIDs → MongoDB ObjectId refs (same purpose, native to Mongo)
  - SQL UNIQUE(user_id, name) on tags → compound unique index { userId, name }
  - item_tags / item_collections are kept as separate junction collections
  (mirrors your SQL join tables) rather than embedding arrays, so they stay
  queryable independently
  - Summary has a unique index on itemId (one summary per item)
  - All enums are enforced via Mongoose's enum validator
